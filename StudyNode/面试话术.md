### 领导是拿来用的，不是拿来供的

“主任，跟您一起工作差不多也有一个多月了。多亏了您的指导和帮助，让我能够快速进步，谢谢您！其实很多活动我也是第一次接触，没那么多经验，但我想做得更好，让您更放心。您看我还有什么需要改进的，或者是做得不够的地方，您都可以告诉我，我也会尽快做出调整，跟上您的节奏。”

利用领导喜欢给人指指点点、好为人师的一面，长此以往，你积极的工作态度展现了好印象，领导对你也更信任，还会给你开小灶。你机会和资源也拿到了，和别人相比已经是next level了。过来人告诉你，打工而已，他就是你play里的一环，仅此而已。

第二招：麻烦领导巩固联系。第一招你没学好，第二招还能帮你大忙。你从现在开始尽快去找你的领导，麻烦——我说的麻烦是请他帮你一个他轻松能完成且不会拒绝你的小忙，然后呢你表达感谢，请他吃个饭，送个小礼物。再有机会啊，找他帮一个大忙，然后送一份大礼。长此以往，经常联系，一个比较牢固且稳定的关系基本就建立了。

我的建议是，找他或她的关系网，擅长什么，请他帮忙。小到咨询，请教一些事情；大到人生大事，比如结婚啊，让他当证婚人等等。我当时是为了买房，就问他有没有这样的资源，刚好他同学在那个楼盘里做销售总监，给我少了几个点。你看啊，一句麻烦让我省了大几万，多了几次礼尚往来，还培养了工作以外的私交。私交可比工作交情更重要，更进一步加深了我们之间的了解和信任，何乐而不为呢？

### 如何做自我介绍
不管你去什么企业面试，面试了多少轮，每一轮面试刚开始，面试官都会让你重新做一个自我介绍。

在做自我介绍之前，有几个注意事项需要我特别强调一下。首先，你要把自我介绍控制在3分钟以内。我在面试其他人的时候发现了一个问题，有80%的同学要么话非常少，要么话非常多。这两种现象都不符合面试官的心理预期，所以这两类人基本没有机会进入到下一轮面试。

我具体跟你形容一下，话很少的同学是怎么做自我介绍的。这类同学的自我介绍通常会在一分钟之内就说完了。先说我是谁，然后介绍自己的毕业学校和所学专业，最后再说一下自己想求职什么样的岗位，然后就坐在那儿等着面试官来提问。作为面试官，我觉得跟这类同学是在尬聊，还得我来启发他，帮他总结是什么性格的人，学习能力如何，技术水平怎么样。作为面试官真的是很辛苦啊，所以遇到这类同学，基本上我都给pass掉了。

各位同学，我们一起来分析一下，为什么有的人在面试的时候话很少呢？我觉得主要有两点原因。首先，这名求职者应该没有做过面试指导，不知道讲什么东西，干脆等到面试的时候看面试官怎么提问吧。其次，这名求职者的简历应该很简单，没有什么可以突出的内容，不敢讲自己的项目经历，怕说漏了不好圆场。很多应届毕业生在求职的时候都会出现话很少的现象。

另外一类学生呢，就是话太多了。自我介绍的时候，知无不言，言无不尽，逮着什么就说什么。比如说自己有什么爱好，平时都愿意做什么，在学校里面篮球打得很好，赢过大学生篮球比赛的团体奖项等等。然后在学生会里，还担任过迎新晚会的男主持人等等。你看他说了这么多，一句有用的都没有，面试官都不知道该问你什么。夸你篮球打得好吗？肯定不行。遇到这样的面试者，基本上我也懒得问什么东西了，礼貌性地打发走人就算了。

大家在做自我介绍的时候，一定要控制在3分钟以内，突出重点。企业招聘的是能干活，能工作的人。你的自我介绍要突出你能干活，而且能跟大家一起干好活儿，告诉面试官你跟这个工作的相关经历。一会儿呢，我会把3分钟的自我介绍拆解成几个部分，每一个部分该突出什么，我会详细地做一个介绍。面试可不是聊天儿，大家要保持严肃感。3分钟的时间里，重点谈跟面试的岗位相关的内容。至于说你自己的喜好以及感情的经历，就不要谈了。

这里呢，我穿插一个小故事。有一次我在IBM面试一个程序员的时候，小伙子刚毕业，有几分朝气。做自我介绍的时候，他跟我提到了个人感情方面的事情。他说自己大学的女同学在我们这工作，他非常喜欢这个女同学，所以就想来IBM求职。各位同学，如果你是面试官，你会给这个小伙子签offer吗？我肯定是不会的。于是呢，我挑了几个比较难的面试题，把这个小伙子给问住了，最后就礼貌性地把他给拒绝了。咱们姑且不谈技术水平如何，就冲着他这个个人感情的因素，我觉得这个小伙子找工作并不理智，没有好好地规划自己的职业生涯，也没有对面试的企业做详细的了解。即便公司把他给招进来了，培养他，还因为点什么事情跟那个女同学闹别扭了，估计这个小伙子一冲动就会提离职。你说企业会把资源用在培养一个感情用事的员工身上吗？当然不会。所以说你在面试的时候，千万别谈自己的感情经历。像什么跟单位女同事恋爱分手了，所以要离开上一家公司之类的事情，千万不要提。还有什么个人兴趣爱好，喜欢旅游，喜欢唱歌，喜欢文学，这些跟工作没有任何关系的事情，千万都不要说了。节省点时间，多谈谈自己的实习经历和项目经验，该多好呢？

有时候我翻阅简历的时候，总能看到，有的同学在简历上写自己参加过哪些社会实践活动，比如在肯德基、星巴克打过工，磨练过自己的意志，锻炼了与人沟通的能力。毕竟程序开发呢，也是需要很多人在一起协作的。所以年轻人参加过一些社会实践活动，对适应工作岗位和融入开发团队会起到一定的作用。但是你把这些写到简历上，得分清主次关系。我建议你把在软件公司实习的经历写在社会实践经历的上面。这样做呢，是让HR更能抓到你简历的重点部分。

做自我介绍的时候，大家不要紧张。你越紧张，语速就越快。你想要突出的地方，面试官还没有反应过来，你就说完了。所以呢，大家要提前把自我介绍的内容写成稿子，然后背熟练了，再对着墙反复练习。熟能生巧嘛。前前后后我面试过的人也超过几百了，我发现有不少人的自我介绍是靠临场发挥的。要么说得磕磕绊绊，要么就是前后思路对接不上。奉劝大家不要幻想面试的时候可以临场抖机灵，真的不管用。你还是老老实实的提前做好准备吧。

介绍自己的时候呢，同学们要注视面试官眼神，不要飘忽不定。我记得有一次我面试一个刚毕业的大学生，他斜着眼睛跟我说话，给我看的难受啊。我就搞不清楚这个同学是从心里鄙视面试官，还是对面试官另有所图呢？面试官如果对一个人的第一印象不好，他不会直接拒绝你，反而会问一些很难的问题，让求职者自己知难而退了。

另外呢，大家在面试的时候不要有小动作。我跟你讲，人在说谎的时候，会不自觉地有一些小动作，比如挠一下脸、撩一下头发、摸一下下巴、攥一下拳头等等。特别是摸鼻子，百分之百会被认定在说谎。所以你在面试过程中，一定要控制好自己的行为，不要有小动作。

接下来呢，我们看一看自我介绍的3分钟里，究竟要讲什么。自我介绍呢，就是要把自我讲明白。对啊，什么是自我呢？在企业的眼里，自我就是学习、工作和沟通的集合。所以大家做自我介绍，要突出自己是怎么学习的，怎么工作的，怎么与人沟通的。那么3分钟的自我介绍可以分解为三个部分。

首先呢，是介绍个人信息的部分。说清楚你叫什么名字，哪一个学校毕业的，所学专业是什么，基本就足够了。这里呢，不要借题发挥。我看有的HR说介绍自己情况的时候可以加上一个小故事，让面试官记住你。比方说呢，有个同学是这么介绍自己的：“面试官，你好，我是左小明，跟左传的左丘明就差了一个字，所以我这个人除了喜欢写程序之外，还很喜欢读历史。”其实呢，这种加入了小故事的自我介绍是毫无必要的。作为面试官，我觉得要么你是在炫耀自己的名字，要么是在卖弄你喜欢读历史，这件事情跟面试求职毫无关联。各位同学一定要记住，自我介绍只讲三点就够了：你是谁，你从哪儿毕业，学的是什么专业，别的一律都不用讲。

那么自我介绍的第二部分呢，就应该讲一下自己的学习经历。有很多同学觉得自己的学习经历很简单，本科学历，计算机

专业，有什么好讲的呢？其实呢，这也是一个重点的部分。你可以讲讲自己学习的课程，比如说：我的本科专业是计算机科学与技术，在校期间学习了数据结构、计算机网络、操作系统、数据库等课程。这些课程是我职业生涯中的基础课程，带给了我扎实的计算机基础知识。即使你学的课程内容并不多，也可以结合自己学习的内容谈一谈你如何去学习这些课程的，这样呢，可以展示你的学习能力。

自我介绍的第三部分呢，就是谈谈自己做过的实习或者项目经历。实习和项目经历中如果涉及到岗位的技能，那就是最好的。如果没涉及到呢，你可以从一些辅助技能入手。比如说在我的简历里，曾经在学校做过一个网站开发的项目，给同学们提供图书借阅的功能。这段经历呢，就让我对项目开发有了更深入的理解。至于说自己在实习的时候，在公司做了什么工作，解决了什么问题，这一段经历就尤为重要了。你可以详细地讲述你做过的项目，和项目有关的技术点，遇到的问题以及解决的方案。这个部分是整个自我介绍中最关键的部分，面试官可以从这里看出你的技术能力和工作能力。

自我介绍讲得很清楚之后呢，接下来就是要回答面试官提出的相关问题了。总结一下，做自我介绍的时候，你的重点一定要放在个人信息、学习经历以及实习和项目经历这三部分。注意控制时间，简洁明了地讲述，不要涉及到无关的内容，比如个人爱好和感情经历。保持自然、自信，不要紧张，也不要有小动作。这样，你的自我介绍才能给面试官留下深刻的印象。大家加油哦！


### 自我介绍范例

上一节课我强调了自我介绍应该说什么。这节课我将以求职者的身份给大家做一个示范，展示怎么用3分钟的时间把自己推销出去。当然啦，如果你的学习经历和项目经历比较丰富，你的自我介绍稍微超过3分钟也是可以的。最重要的是，你要把自己身上的闪光点清晰地传达给面试官才行。

各位同学，接下来我以应届生的身份来做一个自我介绍，大家先听一听我讲得怎么样。

面试官您好，我是中国某某大学的应届毕业生，学的是软件工程专业。我觉得编程技术是学无止境的，仅仅靠学校的专业课所学的内容，远远达不到项目开发的要求。因此，在大学期间，除了学校讲的 Java 基础部分之外，我还自学了 Java 体系中的其他技术，比如说 Spring MVC、MyBatis。包括我还自学了大数据中的 Hadoop 技术。做本校创新学院项目时，我又自学了 Elasticsearch 技术、Python 爬虫技术，还自学了 HTML5、CSS、JavaScript、jQuery、Vue、Bootstrap 等前端技术。

我做过的第一个项目是创新学院的便利店微商小程序。虽然淘宝把零售业整合到了互联网上面去销售，我们买东西很方便，但是学校周围的便利店却不适合借助第三方物流的淘宝模式。所以我们本校的创新学院开发了一个便利店微商的小程序。购物体验类似于饿了么，客户下单之后，便利店会让勤工俭学的同学去送货，最后完成这一单交易。这个小程序我们用的是Uni-App框架，目前只开发了微信小程序，将来会开发支付宝小程序。小程序的后台我们采用的是 GitHub 上面的开源框架，这个框架中整合了 Spring、MyBatis、JWT、Swagger 等技术。这个微商小程序的购物模块和移动支付模块是我开发的。在开发过程中，我遇到的最大问题就是多并发时扣减库存总是出错，不能满足幂等性。后来，我查阅了很多资料，给数据库添加了乐观锁，然后在秒杀促销时配合 REDIS 的事务机制，就再也没有出现扣减库存的 bug 了。

我做的第二个项目是维护本校的学籍系统，并且开发一套新的学籍系统。这个项目也是创新学院的工程。我们学校的学籍系统因为开发较早，用的还是 Struts 加 Hibernate，甚至页面用的是 JSP 技术。现在新的学籍系统要做成前后端分离的架构，可以支持浏览器和小程序的访问。做这个系统最困难的就是新旧系统的并行开发，一边要维护老系统的运行，另一方面还要开发新系统，而且两个系统的数据库和表结构都不一样。我们分成了两个小组，第一小组负责维护老系统，我是第二小组的，主要工作就是开发新的学籍系统。因为学籍信息非常重要，所以我们采用了并行转换的方法，我们写了一组程序在老系统上执行，每一个业务都会在新系统上再执行一次，然后对比新旧系统的业务数据是否一致。新旧系统并行运行了大概半年多的时间，最后才彻底切换到新的学籍系统上。

我觉得参与实际项目的开发，对自己编程能力的提升帮助很大，技术视野也变得开阔了。发现问题和解决问题的责任感和成就感，都是在大学课堂里体会不到的。因此，我希望将来能有一份更有挑战的开发工作，这就是我来应聘某某岗位开发工程师的原因。

听了刚才这位应届生的自我介绍，大家还都满意吧？虽然时间稍微超过了3分钟，但技术亮点非常多，所以这样的自我介绍非常讨面试官的喜欢。作为应届生，做过的绝大多数项目都是面向 CRUD 的，就是对数据库的增删改查操作。那么这种操作大家怎么能说出技术亮点呢？介绍自己的第二个项目时，是一个学籍系统。看似平淡无奇，但他告诉面试官，这个学籍系统是新旧系统并行开发，既要维护老系统，又要开发新系统，而且还要保证新旧系统的完美切换。我相信大家都听出来其中的技术难点了。我们不用讲这个项目有多少个模块，业务流程是什么样的，无论你怎么卖力地介绍这个系统，还是增删改查四个功能。因此，正确的办法是给普通项目加入运维的内容，也就是说用运维技术来凸显你的技术水平。为什么运维有技术含量呢？这是因为业务系统上线之后，我们尽量在业务系统不停机的情况下上线新的功能。李翔同学的新旧学籍系统并行转换，属于灰度发布。当你工作之后，每逢系统升级都会采用灰度发布，一方面让自己的项目经历显得更加真实，另外一方面让面试官看到这个学籍系统虽然技术含量不高，但开发与发布的环节非常严谨，为新旧系统的交接做了大量自动化的调试工作，技术水平应该是不成问题的。我愿意给他一个成长的平台。

那么各位在校的同学，毕竟每个人做过的模拟项目都不一样。我做的这个自我介绍的示范只是举了一个小例子。你只要掌握了这个思路，并且结合你做过的模拟项目，用一两天的时间先打一个草稿，直到修改满意了，你再把它背下来，反复对着墙练习介绍自己。我相信面试官都会被你的自我介绍所打动的。

接下来，我以两年工作经验的程序员为例子，给大家做一个自我介绍。收看我们这个课程的同学，有一部分是有开发经验的，所以你听一下有经验的开发者是怎么做自我介绍的。

面试官您好，我是中国某某大学毕业，学的是软件工程专业，有两年的开发经验。我重点说一下我的项目经历。我第一份参与开发的项目是在北京某公司实习的时候，做的是本地 K12 教育的 App，这个 App 叫做某某名字，您可以在手机软件商店里找到这个 App。我们做的这个 K12 在线教育的 App 的核心功能是直播课程与录播课程。我负责的就是这个 App 的后台部分，开发的是录播课程的模块。为了保护录播视频的版权，我们用的是腾讯云的点播服务，并且做了防盗链和防盗播的功能。用户没有办法在 App 上发录视频，即便把视频的 URL 地址分发出去，别人也是收看不了这个视频的。这个 App 项目用的是前后端分离的架构，前端有 App 和 VUE，后台用的是 Spring MVC 和 MyBatis。因为我实习做的是 App 项目，对这方面还是比较了解的，所以毕业之后就去了北京某某公司。这家公司做的是二手车交易的 App，现在这个 App 已经是中国第二大二手车交易的 App 了。那么这个 App 是 C2C 的平台。在这家公司工作两年多的时间里，我一直都是在做这个二手车交易项目的后台开发，担任软件工程师。我参与做过的模块比较多，比如说有贷款模块、车辆保险模块、分期购车模块、广告投放模块等等。我参与这个 App 的开发，最大的收获就是亲身经历了后台架构的演进，真的是大刀阔斧的改革。最开始这个 App 的后台用的是 Spring MVC 加 MyBatis 做的，后来日活用户到了 60 万的时候，就搞了负载均衡，后台数据库也做了集群，用 Spring Data 和 Redis 对业务数据做了缓存。随着这个 App 的功能越来越多，项目越来越庞大，无法准确地对某一个模块进行优化，并且分配硬件资源。比方说内容推荐模块和保险模块的负载就不一样。如果是一个单体项目，就没有办法单独优化一个模块。所以我们的架构师就把后台的一些模块分离出去了，这样就可以更好地针对某一个业务或者模块单独优化它的性能，所以就搞了微服务的架构。也不是我们开发部门自己在做项目重构，包括 DBA 团队，把数据库也做了垂直切分和水平切分。拆解业务的过程里，我对系统架构有了重新的认识。因为在重构系统的时候，我们先用开源中间件，把数据导入到新的数据平台，然后原系统上的每一笔操作都会自动在新系统上重做一遍，最后再去核对新旧系统数据是否一致。微服务之间的调用，我们还加入了熔断、限流、降级这些措施。从去年开始到现在的一年多时间里，我们这个后台系统一直在向微服务架构演进。我看到贵公司招聘的 Java 开发岗位特别强调要求 Spring Cloud 的经验，我觉得自己还是有这方面经验的，和这个岗位比较匹配。我也希望能在贵公司这样的大平台上继续锤炼技术，提高自己。

相信大家听了这个有两年开发经验的程序员的自我介绍之后，也有了比较清晰的方向。和应届生的自我介绍一样，有经验的程序员在做自我介绍时也要清晰地总结出自己的技术亮点。最重要的一点是面试官在面试时要特别注意，如果一个有经验的程序员仅仅只提到自己做了哪些模块，那就不是很令人满意。必须要强调业务的复杂程度和技术难点，这样才能引起面试官的重视。我相信面试官在接下来听到你们的自我介绍时，会有更清晰的技术判断标准和方向。

好了，以上就是我给大家做的两份自我介绍示范。希望大家能够多多练习，用合适的方式展现自己。如果你们有其他疑问或者想法，欢迎留言给我，我们一起来探讨。谢谢大家！

### 4-3 甄选HR面试核心题库（上篇） 是时候展现职商了】

因为在大厂的面试分为五轮，第一轮和第二轮面试都是偏重于基础知识的，所以大家可以通过刷笔试题巩固一下自己的基础知识。第三轮和第四轮面试偏重于项目经验，所以第二章的前两节课，我就跟同学们说了一下第三轮和第四轮面试应该怎么去做自我介绍。由于不同的项目提问的问题也不相同，所以在后续的章节，我会挑选几个常见的项目方案给大家说一下面试中常见的技术问题应该如何回答。

这节课咱们来看一下第五轮面试，也就是HR面试这一关应该怎么攻克。有很多人以为第五轮面试可算不用谈技术了，于是就放松了警惕。HR有一票否决权，即便你前四轮都通过了，但是HR觉得你不适合应聘的岗位，你还是进不了大厂。

这节课呢，咱们就来刷一下第五轮面试的题库吧。在第五轮面试里边，HR问的问题大致可以分为六大类，分别是个人情况类的问题、个人能力类的问题、工作经历类的问题、进攻性的问题、协聊式的问题、还有结束性的问题。这六大类的问题一节课的时间肯定讲不完，所以我要分成几节课来把这些问题讲清楚。

我们先来看一下个人情况类的问题都有哪些。我一共列出来HR常问的七个问题，因为PPT的篇幅有限，所以我把这七个问题的答案整理成文档，然后以锚点链接的方式弹出来，各位同学可以参考一下。

比如说呢，咱们先来看一下第一个问题。HR说了，请你先做一个自我介绍。各位同学还是按照我教给你的方式重新说一下自我介绍。

那么下面的第二个问题是：你认为自己最大的优点是什么？你可以这样回答：我认为我的优点可以分为三个方面。第一个优点是头脑清醒，思路清晰，遇到事情沉着冷静，能分清事情的轻重缓急。我的第二个优点是工作有计划，有条理，能积极主动做一些需要做的事情，而不是等着别人要求你去做什么。我的第三个优点是适应能力强，而且求知欲也很强，能够不断学习上进。

接下来呢，我们去看一下第三个问题：你认为自己最大的缺点是什么？这个问题就更不好回答了。如果说深了呢，会让面试官觉得你这个人不太行。假设你跟HR说自己小心眼、爱嫉妒、人非常懒、而且脾气还很大、工作效率又低，那么企业肯定不会录用。你要是说浅了呢，又会让HR觉得你自己不够深入、客观地面对自己的缺点。大家绝对不要自作聪明地回答“我最大的缺点就是过于追求完美”。有的人以为这样的回答会显得自己比较出色，但事实上呢，这已经很危险了。企业喜欢求职者从自己的优点说起，中间加一些小缺点，最后把话题又转回到优点上，突出优点的部分。企业喜欢聪明的求职者。

比方说呢，你可以这么去回答：“我性子急，别人都嫌我督促工作的时候催得太紧。如果有些事情非常紧急，我嫌别人做得又太慢，我就替别人把事情先给做了。”

接下来呢，咱们看一下第四个问题是什么：最能概括你自己的三个词是什么？这道题的标准答案是这样的：诚实可信、适应能力强、而且敢于担当。

接下来呢，咱们再看一下第五道问题是怎么说的：HR是想了解一下你跟同事之间的关系。你可以这么去说：“我的同事都认为我是一个动手能力强、做事有始有终的人。其实有很多事情我喜欢自己去验证答案，所以我很愿意亲自动手去做自己想做的事情。一旦我做了，我也会把它做到最好。我认为在做每一件事情的过程中可以收获非常多的东西。”

我们再来看一下第六个问题：你最近5年的职业规划是什么？这个问题最忌讳回答的就是“我不知道，我没有想过”。企业都喜欢有进取心的人，不一定偏要回答想要做管理岗位，毕竟一个公司的管理岗位数量还是很少的。所以你可以立足于你自己的技术长项来规划一下自己的职业发展。

比如说：“我现在更多的是运用后台的框架和技术，那么在未来的5年里，我希望我能更深入地了解本公司所用的框架。我想先成为全站工程师那样的多面手，然后有能力参与架构的设计。我最终的目标是想成为系统架构师。”各位同学可以根据我这个示范做一些修改，把自己追求技术而且积极进取的那一面给凸显出来。

那么接下来呢，我们去看一下第七个问题是怎么问的：你从上一家公司离职的原因是什么？这个问题有很多的忌讳，同学们千万不要说上一家公司的坏话，比如说跟领导不和、管理混乱、人际关系复杂、公司不注重人才、薪资待遇给得不够，这些啊通通都不要说。就算你在前一个公司受到再大的委屈，对公司有多少怨言，千万都不要表现出来。尤其要避免对公司领导的批评，避免面试官产生负面情绪。那么建议呢，这个时候最好的回答方式就是把问题归咎在自己的身上，找一些不会损害自己能力的理由。

比如说：“我搬家了，距离上一家公司太远了，所以我想找一个离自己住的地方近一些的公司。”再比如说：“我家是青岛的，但是我女朋友是北京人，她在北京找到工作了，所以我就从青岛的那家实习公司离职了，来北京定居，找一份开发的工作。”你看，这样的回答就特别好，把离职原因说成了不可抗拒的因素，又没有批评上一家公司，这个回答是非常完美的。

那么接着第七个问题，我再往下补充一下。如果HR看到你的户口不是本地的，他会下意识地问一下你为什么要留在北京或者上海工作。如果你回答北京或者上海工作机会多、工资待遇高，虽然这是你心里的实际想法，但是HR会觉得如果将来深圳有适合你的工作，待遇比现在还要好，你是不是会离职去深圳发展呢？那公司岂不是白白培养你了吗？遇到这个问题，你可以从很多角度来回答。例如，女朋友在北京的男同学，你可以说：“我的女朋友在北京找到了一份很满意的工作，所以呢我也打算留在北京。而且我学的是计算机相关专业，北京有这么多软件企业，IT行业发展的这么好，我肯定是要留在北京发展。”如果没有女朋友的男同学，你可以从别的角度去回答这个问题。

比如说：“我老家是一个三线的小城市，我学的是计算机专业，回到老家肯定找不到合适的工作。对于我来说，工作机会最大的城市就是北京，所以我想留在北京发展。而且呢我以前在北京生活过一小段时间，我可以融入到这个城市。”那么回答这道问题，我们还可以换一个角度。以前呢我教过一个女同学，她老家是广西的，她就想在广州或者深圳找一份工作，这样可以离自己的老家近一点，而且广东、广西语言和风俗上比较接近。最后这个女同学如愿以偿，进入了深圳的腾讯公司。当然了，每个同学留在一个城市的原因可能都不会相同，也不是说大家都要留在北京工作，上海、杭州、深圳这些城市的IT行业发展的也很好。所以大家结合自身的情况，找一个合适的角度去回答这个问题。

接下来呢，咱们看一下个人能力类的问题有哪些。第一个问题是：你怎么理解你应聘的职位？你可以这么去回答：“我应聘的是Java软件工程师，因为之前我在上一家公司做过电商系统，所以我了解这个岗位的开发工作都有哪些。比如说参与需求分析、项目评审，根据电商系统的业务特点完成程序的开发工作。除了我们自己写一些单体测试，还要协助测试人员完成集成测试和系统测试。公司有新入职的员工，我有时候还要去指导新员工以及实习生等等。”

那么接下来我们看一下第二个问题是怎么说的：如果让你管理一个小团队，你应该怎么去管理呢？你可以这么去说：“如果是管理新入职的实习生或者应届生，我会采用家长式的管理模式。因为很多实习生和应届生啊，不知道自己适合做什么，能

做什么，工作方法和效率也不高，所以我会采用家长式的管理方式，多去关心他们、指导他们，帮助他们去找适合自己的岗位，让他们对工作和生活都有一个明确的目标。而对于已经在公司工作了一段时间的员工来说呢，我会采用民主化的管理模式。比如说安排某些工作的过程中，我会经常性地跟组员进行讨论，让大家各抒己见，发挥每个人的主观能动性。实在遇到有争议的地方，我们还可以投票表决，少数服从多数，这样可以避免一些同事对我的管理不服气的问题。”

接下来我们看一下第三个问题是怎么说的：你怎么去解决一些困难的问题？你可以这么去说：“当我在工作中遇到一些困难的问题的时候，我会把这些问题拆分开来，各个击破。比如说开发一套分布式系统，首先我要从架构上去设计，确保系统的健壮性。在开发过程中，我会先编写框架代码，确保每个模块都能正确调用，然后去编写每个模块的代码，确保业务代码的逻辑正确。开发完成后呢，我会去写一些单体测试用例来测试每个模块的功能，再去写一些集成测试用例来测试模块之间的接口，最后写一些系统测试用例，来测试整个分布式系统的性能、负载均衡等方面的性能。”这样回答呢就显得你特别有条理。

接下来我们看一下第四个问题是怎么问的：在上一家公司，你做过的最有成就感的一件事是什么？你可以这么去说：“在上一家公司，我参与了一个大型的电商系统开发。这个系统呢不仅仅是要支持多个网站、多种业务、还有多种支付方式，还要支持高并发的访问量。最终，我和我的团队通过技术攻关，成功地开发出了一套高性能的分布式电商系统，系统上线后运行稳定，得到了客户和公司的高度评价。这是我在上一家公司做过的最有成就感的一件事。”

接下来呢，我们去看一下第五个问题是怎么问的：你认为你在团队中扮演什么角色？你可以这么去说：“在团队中，我认为我扮演的是一个协调者的角色。我的性格比较随和，能够与团队中的每一个成员相处融洽。当团队中出现分歧的时候，我会积极沟通，协调各方的意见，找到一个折中的方案，确保团队的合作能够顺利进行。而且呢，我也会主动承担一些繁琐的工作，为团队中的其他成员分担压力。”

那么接下来我们再去看一下第六个问题是怎么问的：你如何看待加班？这个问题啊很敏感，回答得好呢，可以加分，回答不好呢，可能会减分。你可以这么去说：“我认为加班是工作中的一部分，但是不应该成为常态。如果有项目的紧急需求，需要加班完成任务，我会毫不犹豫地去加班，确保项目按时交付。但是我也认为，长时间的加班会影响工作效率和身体健康，所以我会在平时的工作中尽量提高工作效率，避免加班。”这样回答呢既表现了你的责任感，又不会让人觉得你是一个只知道加班的工作狂。

最后呢，我们再看一下第七个问题是怎么问的：你怎么处理工作中的压力？你可以这么去说：“工作中的压力是不可避免的，但我认为压力也是一种动力。当我感到压力很大的时候，我会先冷静下来，把手头的工作按轻重缓急排个优先级，逐步去解决。平时呢，我也会通过运动、听音乐等方式来放松自己，保持良好的心态。”这样回答呢既表现了你的抗压能力，又不会让人觉得你是一个被压力压垮的人。

### 4-4 甄选HR面试核心题库（下篇） 是时候一展情商了】

HR常问的第四类问题是进攻性的提问，一共有四道问题。咱们先从第一道问题开始说。

第一道问题是这么说的：“你对我们公司了解有多少呢？”如果求职者回答不上来，就说明这个人是海投的简历，根本没有认真了解公司的背景和工作岗位的要求，自己都没有先审视一下是否跟这家公司和应聘的岗位匹配，那么HR也不会帮你做决定的。所以海投简历的人很多都被pass掉了。你去面试之前一定要先了解这家公司的业务背景和你应聘岗位的具体要求。所以这道题呢，大家可以按照百度百科上面的介绍讲一下这家公司的具体业务。

那么第二道问题是：“你喜欢这份工作的哪一点？”每个人的价值观是不同的，自然评判的标准也会不同。就算你最看重的是这个岗位的薪资待遇，但是你绝对不能把这个想法透露出来。你应该围绕自己的技术爱好来说。你可以这样去回答：“我实习和工作期间做的都是APP后台开发，在这方面积累了大量的经验，这个岗位能让我充分发挥才能，所以我才想找一份类似的工作。”

好了，咱们看一下第三道题是怎么问的：“你能为我们带来什么？”企业很想知道新员工能为公司做什么，求职者应该重申自己的优势。因为你的工作经历并不是太多，所以你不用强调自己的业绩，比如说让公司的日活用户提升了多少，或者说给公司挽回了多少损失等等。你就强调你可以做好本职工作就够了。你可以这样去回答：“我实习和工作期间做的都是APP后台的开发，这个领域我比较擅长，所以我能在新的岗位上发挥我的才能，保证APP新业务的按时上线，并且维护好现有的模块。”

接下来我们看一下第四道问题是怎么说的：“你申请的这个岗位，你认为你还欠缺什么？有哪些可以预见的困难？”企业喜欢问求职者的弱点，但精明的求职者一般不直接回答。企业希望看到求职者重申自己的优势。你可以这样去回答：“对于这个职位和我的能力来说，我相信自己是可以胜任的。如果遇到技术难题，我会请教团队里的前辈。而且我的学习能力很强，我相信可以很快融入这家公司的企业文化，进入到工作状态。”

接下来我们看一下闲聊式的六个问题。

第一个问题是：“你有什么业余爱好？”面试官的时间都是很宝贵的，不可能会问一些没有目的性的问题。HR是想通过求职者的业余爱好来了解你的个性、品德、喜好与心态等方面的情况，借此从侧面看出求职者的人格结构和生活的丰富性。别忘了你是在面试，不好的爱好和不符合职业需求的爱好是不能说的。你可以这样去讲：“我喜欢在工作之余浏览技术网站，并且写技术博客，把我解决的技术问题总结成博客文章，总结的过程就是重新复习了一遍当初解决问题的方案。所以这些年，我都是通过这种方式来巩固自己的技术。另外，我平时喜欢研究树莓派这种物联网设备，写一些Python程序来控制硬件设备，觉得挺有意思的。”

往下我们看一下第二道问题是怎么说的：“你最喜欢的书籍是什么？”大家尽量说本专业内的书籍。比如说：“我喜欢看《人月神话》这本书，因为这本书介绍了大量工程实践的经验。还有《Effective Java》这本书，作者写了Java API，一共提到了57条Java程序的优化经验，我觉得非常有意义。还有《设计模式》这本书，设计模式几乎成了系统架构师必读的内容，23种设计模式真的太棒了，高内聚、低耦合的设计思想应该贯穿程序开发的始终。其他的书籍我也看了很多，只是觉得以上这些书籍最有意义。”

接下来我们看第三道问题是怎么说的：“你认为什么是成功？”成功就是达成了所设定的目标。成功是一种积极的感觉，它是每个人达到自己理想之后的一种自信状态和满足的感觉。如果找到成功的规律，成功就变成了非常简单的事情。成功是没有捷径的，要有强烈的欲望，付出比别人更多的努力。

那么接下来我们看一下第四道问题是怎么说的：“你认为什么是失败？”失败就是没有达到预期的目的，或者输给了对方。失败也是一种感觉，一种绝望的感觉。失败并不可怕，关键是对待失败的认识和态度。失败是对弱者的一次打击，对强者却是一次激励。无数人的失败都是败给了做事不彻底，往往做到离成功只差一步就停下来了。

接下来我们看一下第五道问题：“谈一谈你对跳槽的看法。”这道问题不太好回答。你可以这样讲：“正常的跳槽可以促进人才的合理流动，应该支持。但是频繁的跳槽对企业和个人都不利，应该反对。现在有很多年轻人一言不合或者受一些挫折就离职跳槽，我反对这种行为。因为我们这个行业有‘35岁中年职业危机’一说，频繁的跳槽不能深入理解一个项目的业务和技术架构，只是做一些表面修修补补的工作，对技能的提升和经验的积累都没有好处。所以频繁跳槽的人到最后只会被这个行业所淘汰。”

接下来我们看一下第六道问题是怎么说的：“你怎样看待学历和能力？”这道问题比较好回答。你可以这样讲：“学历虽然不是最重要的，却也是一个人学习能力的体现。虽然不能直接和工作能力划等号，但是如果一个人的学习能力很好，他的接受能力也会比较强，更能快速适应工作。但是起绝对作用的还是一个人对工作的热情和刻苦的程度。”

最后我们看一下结束性提问的六个问题。

第一个问题是：“如果企业录用你，你应该如何展开工作？”你可以这样回答：“我先了解企业的规章制度，严格遵守公司的纪律，学习项目里的知识和技术以及作业流程。在工作中，我会按照项目的流程去工作，工作之余，我会与同事多交流，多学习大家在工作上的窍门，并且我会规范自己的工作流程，缩短自己与同事在工作上的差距。”

接下来我们看一下第二个问题：“你能在我们公司干多久？”几乎没有面试官会相信“永远”二字，这是一个不切实际的答案。其实HR不在乎你的回答中说要在公司工作多长时间，企业在乎的是你在公司现阶段的态度。你可以这样讲：“直到我无法再接受新事物，并且公司也认为我不适合这个岗位了，我就会选择离开。只要我在岗位上工作一天，我就要承担一天的责任，为公司创造一天的效益。”

接下来我们看一下第三个问题是怎么说的：“工作中你不适合这个岗位，你应该怎么办呢？”你可以这样讲：“如果差距较小，能通过刻苦学习或者更多的付出可以适应这个岗位，我会付出更多的努力。如果根本不适合，差距较大，我会立即离开，换一个环境。这样对单位和个人都有好处。”

接下来我们看第四个问题是怎么说的：“你对薪资的要求是什么？”这道问题有两种回答。我先说第一种回答。你可以直接讲明自己对薪资的要求。需要注意的是如此理直气壮地回答的前提是你有谈薪水的资本，也就是说你确实能够胜任这份工作，你的简历内容很丰富，你在前一家公司做出了很好的成绩。你用自己的实力说话才能谈到高薪。

另外一种回答呢，就是多讲大道理和职业的远景。这样的回答适用于大多数选择跳槽但简历又没有亮点的求职者。谈薪资的时候不要怕说具体的数字，只要不说得太高，基本上符合这个岗位的平均工资或者稍微偏上一点点即可。即便你说的工资稍微高了一些，如果HR很看好你，他会直接跟你说：“目前来说，这个岗位还满足不了你的薪资要求，我们能给出的薪资是这个数字，你能接受吗？”如果你觉得可以，那就接受。如果不能接受就拒绝。如果企业根本不看好你，HR是不会给你讲这个岗位的工资上限的。所以你可以放心说具体的数字，但还是要保守一点。你可以这样说：“跟薪资相比，我更看重的是职业发展。我相信贵公司在处理这个问题上能根据我的实际情况和现在的市场行情给我一个合理的

薪水。目前我期望的是能达到年薪8万这个水平。”

接下来我们看一下第五道题是怎么说的：“你还有什么要问的吗？”HR的这个问题看上去是可有可无的，其实还是很关键的。企业不喜欢说“没有问题”的人，因为公司很看重员工的个性和创新的能力。企业不喜欢求职者问一些个人福利之类的问题。如果你这样问就非常好：“贵公司对新入职的员工有什么培训项目？我可以参加吗？”或者：“贵公司的晋升机制是什么样的？”因为这样的问题体现出你对学习的热情和对公司的忠诚度以及你的上进心，企业会非常欢迎这样的员工。

最后一个问题是：“你还有什么需要我们知道的吗？”也许应聘者觉得没有什么可说的了，但这是一个事半功倍的极好机会，要是放弃了就太可惜了。你可以这样跟HR说：“我们谈得很详细了，但我还要强调一下我有能力也有信心来胜任这个岗位，我认为可以为公司做出积极的贡献。”

好了，各位同学，结束性的六个问题，包括HR的题库，我们就全都刷完了。看到这么多HR的问题，很多应届生现在是不是脊背发凉呢？如果不是咱们提前看了这些问题，要是真的到第五轮HR面试，恐怕很多同学会被一票否决。现在我们已经知道HR要问哪些问题了，如果在面试中遇到HR即兴问一些问题，你也不要慌张，说一些政治正确的大道理就行了，既不会加分，也不会丢分。

# 面试官完全拒绝不了的自我介绍是什么样的

很多人的自我介绍是这样的：“你好，我叫某某某，我是哪里人，我从哪个学校毕业的，我学什么专业的，某年某月某日在某家公司工作，做的是什么。” 为什么这个自我介绍差呢？因为没有提供信息增量。自我介绍是一个特别好的推销自己的机会，你要把自己的优势和亮点都展示出来，千万不要把时间浪费在重复简历上的话。如果面试官扫一眼你的简历都不愿意，这样的公司你不去也罢。

接下来，我要讲适合老手和新手的实战案例。这个视频有点长，请耐心看完，真用到的时候你就赚大了。

现在我来演示一下老手的自我介绍。计时开始：

“我叫玛丽，针对贵公司副总裁的岗位，我有三点优势。第一，我有丰富的产品和业务经验，曾经在李开复老师的创新工场工作，我做的项目被阿里收购了，之后还获得了几千万的知名基金以及大厂的投资。第二点，我看到咱们岗位描述里特别看重用户增长的经验，我从0到1系统地操盘过。第三，在搭建和管理团队方面，无论是大中小型团队，我都带过，特别擅长招聘和培养人才，可以快速让团队形成战斗力。请问你们对这个岗位还有什么样的要求吗？我会根据这些要求坦诚地分享自己的经历，看看我们是否匹配。”

有人会说：“你这个经历我们模仿不了啊。” 别着急，下面我要讲职场新人怎么办。

首先，思考一下我刚刚的自我介绍，你会发现其实是有框架的。第一，我先说的是整体优势，比如产品和业务的经验。第二，我说的是这个岗位最看重的重点优势，比如用户增长的能力。第三，我说的是这个岗位必须具备的通用能力，比如管理能力。最后，我还请面试官说一下这个岗位其他的要求，为下一步的沟通留了个尾巴。

注意这个框架模板：岗位需求 + 整体优势 + 岗位重点优势 + 岗位托底优势 + 后续沟通影子。把这个视频收藏一下，以后你一定用得到。

现在我来带你练一练，看看这个框架如何实际应用。假设一个完全没有实践经验、想要转行做运营的人，可以这样说：

“我叫马小丽，对于运营这个岗位，我有三点优势。第一，我认真学习过运营的基本方法和流程，具备常用的社群运营、用户运营的能力。第二，做用户运营时特别需要沟通能力，这方面我非常擅长，有过某些案例。第三，我的学习能力和自我驱动能力都很强，哪怕没有经验，我也会积极学习。例如，来之前我专门分析了贵公司的运营流程，找出了一些可能存在的问题。如果您感兴趣，我很希望分享出来供您参考，并请您指导。”

你看我的自我介绍是不是按照这个框架来的？在这个例子里，最大的特点是这个同学虽然没有实际工作经验，但按照我说的框架，他依然能够讲出自己的优势亮点。

我在北邮给研究生讲课时，发现很多学生，甚至是一些工作了几年的，都不知道怎么做自我介绍。第一印象太重要了，机会往往就是这么丢掉的。我最早的学生都已经是腾讯的总监了。按照我说的来，绝不会有错的。

来，我们再复习一遍这个框架：岗位需求 + 整体优势 + 岗位重点优势 + 岗位托底优势 + 后续沟通影子。你可以用这个框架把这两个自我介绍对照拆解一下。我分享的这些框架，都是我自己总结的。我挺讨厌那些现在动不动抄来抄去的人，源头在我这儿呢！大家能不能多鼓励鼓励我，给这个视频点点赞，关注一下，我的动力就更足了。


# 面试自我介绍，有的人面试一开口就赢了
好的自我介绍在面试中至关重要，今天带你完整掌握如何通过自我介绍打动面试官。我们推荐采用“总—分—总”的结构，这样可以让你的表述更加清晰、有条理。

### 1. 总述：一句话精准介绍自己
自我介绍的开场需要瞬间吸引面试官的注意，一句话应当精准融合你的核心专业经验和行业经验，呈现出你的行业标签和专业标签。

**例子：**
- “我是一位在教育领域深耕了10年，专注于课程研发和教学管理的专业人士。”
- “我拥有5年互联网行业营销经验，擅长数据驱动的市场策略制定。”

通过这样一句话，面试官能够迅速对你的职业水平和行业背景有一个初步且准确的认知。

### 2. 分述：详述核心经历及产出
在这一部分，你需要通过分段描述你的核心经历及产出，展现你的价值。可以用一个由远到近的成长故事来展示你的职业发展。

**例子：**
- **初期经历：** 刚毕业时，你可能在某个基层岗位默默耕耘，积累了宝贵的基础经验。例如，你在A公司从事市场营销工作，最初几年深入学习市场调研方法，掌握了与客户初步沟通的技巧。你可以提及在A公司参与的某个项目，讲述如何通过自己的努力为项目带来一定的成果，哪怕这个成果在当时看起来不大，但却是你成长的基石。

- **中期发展：** 随着经验积累，你可能进入了一家更具规模的企业，开始独当一面。例如，在B公司，你负责了一个重要的市场推广活动，通过创新策略和高效执行，使产品知名度大幅提升，市场份额也显著增加。在这里，要详细描述你采取的行动、遇到的困难以及最终的成绩，让面试官看到你的能力提升和价值创造。

- **近期成就：** 到了近期，你可能在一家知名企业担任关键角色，带领团队取得了突破性的进展。例如，在C公司，你已经是团队负责人，成功策划并执行了一系列大型营销活动，不仅提升了品牌形象，还实现了业绩翻倍增长。要分享在此过程中获得的领导力、团队协同以及战略规划等宝贵经验。

**要点：**
- **明线与暗线：** 讲述工作的表面内容和成果（明线）的同时，也揭示背后的职业选择动机（暗线）。例如，你从A公司跳槽到B公司，可能是因为看到了行业发展的新趋势，渴望接触更前沿的技术和理念。通过讲述这些暗线，让面试官了解你是一个有清晰职业规划、积极进取且适应变化的人。

- **撒种子：** 在适当时候撒种子。例如，你应聘的公司正在进行海外市场拓展，而你过去的工作中有过相关国际项目的经验。可以在介绍中提到：“我曾主导过一个海外市场的开拓项目，如果您感兴趣的话，稍后我可以分享。” 这样的话，能够激发面试官的好奇心，为后续深入交流埋下伏笔。

### 3. 总结：总结核心优势，表明求职动机
最后，简要总结你的核心优势，并表明你的求职动机。

**例子：**
- “我具备丰富的行业经验、敏锐的市场洞察能力以及卓越的团队领导能力。由于上家公司因业务发展影响持续减员，我上个月主动离职，现在期待能与贵公司一起创造更大的价值。”

### 结语
采用“总—分—总”的结构，在短短几分钟内，你就能全面而深入地展示个人能力、经验和潜力，在众多面试者中脱颖而出。祝你能拿到理想的offer，加油！
