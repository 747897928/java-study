## 1. 先怎么用这份手册

你接下来不要只做“刷题”，而要做两件事：

1. 学会从暴力法推出优化法。
2. 把“题型识别信号”练成条件反射。

复习时建议固定顺序：

1. 先看题目在问什么。
2. 先自己说出一个暴力法。
3. 再问自己：哪里在重复计算？
4. 再问自己：相邻状态之间是不是只差一个元素 / 一行 / 一列？
5. 最后才看优化法和代码。

如果你直接背优化法，换个题面就容易不会。
如果你能稳定从暴力法推出优化法，换题你也能继续做。

## 3. 前缀和方法论

### 3.1 什么叫前缀和

前缀和的本质不是“背一个公式”，而是：

把“重复计算前面那一大段和”的工作，变成“复用上一次结果”。

最经典定义：

- `prefix[i]` 表示前 `i` 个元素的和
- 也就是 `nums[0..i-1]` 的和

于是区间和可以写成：

```text
sum(left, right) = prefix[right + 1] - prefix[left]
```

### 3.2 什么时候应该想到前缀和

看到下面这些关键词，优先想到前缀和：

- 左半部分和右半部分
- 上半部分和下半部分
- 是否存在一个分割点
- 很多次区间求和
- 统计某类连续子数组
- 枚举每个位置时都在重复算“前面那一坨”

一句话记忆：

`如果重复计算的是“前缀区域”的和，就优先想前缀和。`

### 3.3 如何从暴力法推到前缀和

不要死背，按这 4 步推：

1. 先把暴力法写出来。
2. 找出重复计算的部分。
3. 看看能不能先求总和。
4. 看看当前答案能不能由“上一个答案 + 一个新元素”得到。

举例：

- 暴力法：每一条切分线都重算上半部分和下半部分和。
- 重复部分：前面很多行 / 前面很多列被反复累加。
- 优化点：先求 `total`，只维护一边的 `prefix`。
- 判断条件：`prefix == total - prefix`，等价于 `2 * prefix == total`。

### 3.4 前缀和常见坑

- 坑 1：求和用 `int` 溢出
  - 只要题目里 `n` 和元素值都可能很大，先考虑 `long`
- 坑 2：忘了初始化 `prefix[0] = 0`
  - 尤其是区间和、哈希表统计时很关键
- 坑 3：把“存在”题写成了“统计”题，或者反过来
  - 题目到底在问是否存在、求最大值、还是统计个数，要先分清
- 坑 4：两边必须非空却枚举了最后一个位置
  - 比如切分题，只能枚举到 `rows - 2` 或 `cols - 2`
- 坑 5：前缀和够用了还去上二维前缀和
  - 只要不是“任意子矩阵和”，通常没必要上二维前缀和

## 4. EqualMatrixPartition 这题要真正学会什么

### 4.1 这题的题型

这题不是“二维前缀和题”，更准确地说，它是：

`枚举分割线 + 前缀和`

甚至可以再直白一点：

`把二维问题压成一维的行前缀和 / 列前缀和问题`

### 4.2 从暴力法怎么推

假设在第 `i` 行下面切开：

- 上半部分和 = `prefixRow`
- 下半部分和 = `total - prefixRow`

如果两边相等：

```text
prefixRow = total - prefixRow
2 * prefixRow = total
```

竖切完全一样。

所以真正需要维护的只有一个量：

- 当前切分线之前的和

### 4.3 为什么推荐写法不是二维前缀和

二维前缀和当然能做，但这里不是最短路径。

因为你根本不需要问“任意子矩阵和”，只问：

- 前若干行的和
- 前若干列的和

这时直接扫行前缀、列前缀更短、更稳、更像考试现场解法。

### 4.4 这题迁移到别的题时要记住什么

你真正要带走的是这句话：

`如果我在枚举每个分割点时，都在重复算某个前缀区域的和，就应该想前缀和。`

## 5. 前缀和 3 题训练

这一组按难度递进：

1. 先练“分割点相等”
2. 再练“多次区间求和”
3. 最后练“前缀和 + 哈希表统计个数”

### 5.1 题 1：寻找数组的中心下标

对应代码：[PivotIndex.java](./src/main/java/com/aquarius/wizard/leetcode/prefixsum/PivotIndex.java)

**题目**

给你一个数组，找一个位置 `i`，使得它左边所有元素之和等于右边所有元素之和。

**暴力怎么想**

- 枚举每个位置 `i`
- 重新计算左边和
- 重新计算右边和
- 看是否相等

时间复杂度 `O(n^2)`。

**优化怎么推**

先求总和 `total`。

扫到位置 `i` 时：

- `leftSum` 表示左边和
- `rightSum = total - leftSum - nums[i]`

于是每个位置只要 `O(1)` 判断。

**你真正要学会的点**

这题和 `EqualMatrixPartition` 的本质是一模一样的：

- 一个位置把整体分成左右两边
- 一边的和可以通过总和减出来

**题型总结**

看到“一个位置把整体切成两边”，先想：

`总和 + 左前缀和`

**举一反三**

- 平衡字符串分割
- 数组分成两部分和最接近
- 某个下标左边条件和右边条件同时满足

### 5.2 题 2：区域和检索 - 数组不可变

对应代码：[RangeSumQueryImmutable.java](./src/main/java/com/aquarius/wizard/leetcode/prefixsum/RangeSumQueryImmutable.java)

**题目**

给定一个数组，后面会进行很多次 `sum(left, right)` 查询。

**暴力怎么想**

每次查询都从 `left` 加到 `right`。

单次 `O(n)`，多次查询会慢。

**优化怎么推**

建前缀和数组：

- `prefix[i] = nums[0..i-1]` 的和

于是：

```text
sum(left, right) = prefix[right + 1] - prefix[left]
```

建表 `O(n)`，查询 `O(1)`。

**你真正要学会的点**

这类题不难，但它会帮你把前缀和公式练熟。

以后只要看到：

- 同一个数组
- 多次区间和查询

就要想到它。

**题型总结**

`静态数组 + 多次区间求和 = 前缀和数组`

**举一反三**

- 二维区域和检索
- 成绩区间统计
- 股票某区间累计收益

### 5.3 题 3：和为 K 的子数组

对应代码：[SubarraySumEqualsK.java](./src/main/java/com/aquarius/wizard/leetcode/prefixsum/SubarraySumEqualsK.java)

**题目**

给你一个数组和整数 `k`，统计和为 `k` 的连续子数组个数。

**暴力怎么想**

- 枚举每个起点
- 向右扩展终点
- 边扩展边累加
- 和等于 `k` 就计数

时间复杂度 `O(n^2)`。

**为什么不能只用普通前缀和数组**

因为这题不是问“某一段区间和是多少”，而是问：

`有多少段区间的和等于 k`

这意味着你在扫到当前位置时，需要知道：

`之前有多少个前缀和刚好等于 当前前缀和 - k`

**优化怎么推**

如果区间 `[j + 1, i]` 的和等于 `k`，那么：

```text
prefix[i] - prefix[j] = k
prefix[j] = prefix[i] - k
```

所以遍历到当前位置时：

- 先看历史上 `prefix - k` 出现过多少次
- 这些出现次数都能和当前位置组成合法子数组

于是需要：

- 前缀和
- 哈希表记录前缀和出现次数

**你真正要学会的点**

当前缀和题从“求值”变成“统计个数”时，常常要加哈希表。

**题型总结**

`统计连续子数组个数` 这类题，经常是：

`前缀和 + 哈希表`

**举一反三**

- 和为 `k` 的子数组个数
- 和能被 `k` 整除的子数组个数
- 0 和 1 数量相同的最长子数组

## 7. SQL 高频模板（可选）

你说 SQL 大概率不考，但模板还是放在这里，后面如果要补可以直接拿来改。

### 7.1 分组统计后筛选

```sql
SELECT department_id, COUNT(*) AS cnt
FROM employees
GROUP BY department_id
HAVING COUNT(*) >= 3;
```

### 7.2 每组取最新一条

```sql
SELECT *
FROM (
    SELECT t.*,
           ROW_NUMBER() OVER (
               PARTITION BY user_id
               ORDER BY created_at DESC
           ) AS rn
    FROM orders t
) x
WHERE x.rn = 1;
```

### 7.3 左连接后做聚合

```sql
SELECT c.customer_id,
       COUNT(o.order_id) AS order_count
FROM customers c
LEFT JOIN orders o
    ON c.customer_id = o.customer_id
GROUP BY c.customer_id;
```

### 7.4 每组 Top N

```sql
SELECT *
FROM (
    SELECT e.*,
           ROW_NUMBER() OVER (
               PARTITION BY department_id
               ORDER BY salary DESC
           ) AS rn
    FROM employees e
) x
WHERE x.rn <= 3;
```

### 7.5 累计和 / 窗口求和

```sql
SELECT user_id,
       created_at,
       amount,
       SUM(amount) OVER (
           PARTITION BY user_id
           ORDER BY created_at
           ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
       ) AS running_amount
FROM payments;
```
