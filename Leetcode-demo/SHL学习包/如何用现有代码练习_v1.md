# 如何用现有代码练习 v1

你现在手里的资产有三层：

1. 英文题面
2. Java 代码
3. 中文注释 / 学习备注

最好的使用顺序不是直接看答案，而是分层使用。

## 推荐流程

### 第一步：只读英文题面

先只看对应类最上面的英文部分：

- `Question`
- `Input`
- `Output`
- `Constraints`
- `Example`

先不要往下看“我的备注”。

你要先自己回答这 4 个问题：

1. 输入到底是什么
2. 输出到底是什么
3. 目标是求最值、可行性、计数，还是构造
4. 约束在暗示什么复杂度

### 第二步：自己写一版

你可以直接：

1. 复制一份类到你自己的练习目录
2. 或者先拿草稿纸 / 脑子里写思路
3. 再自己写 `solve` / `main`

重点不是一次写对，而是先暴露自己的第一反应。

### 第三步：再看中文注释

这时候再去看：

- 难度
- 考点
- 校对
- 提示
- 方法上的长注释

这一步的目标不是“看懂答案”，而是比较：

1. 你和现有解法，第一反应哪里不同
2. 现有解法抓住了什么你没抓住的结构
3. 下次遇到相似题时，你能不能更快识别模式

### 第四步：写自己的复盘

每做完一道题，只记 3 句就够：

1. 这题的本质是什么
2. 我一开始想错在哪里
3. 下次看到什么关键词，我会想到这题

## 最容易犯的错误

### 1. 太早看中文解释

这样会让你误以为自己会了，其实只是看懂了别人的推导。

### 2. 只记代码，不记建模

真正决定你考试能不能迁移的，不是代码细节，而是：

- 为什么这是滑动窗口
- 为什么这是函数图
- 为什么这是树边贡献
- 为什么这是“枚举一边、解另一边”

### 3. 想一口气刷太多

你现在时间不够。

比起“每天碰 3 道，没一道吃透”，
更有效的是“每天吃透 1 道，周末再补一题相似题”。

## 什么时候看 main

`main` 的作用主要是：

1. 帮你把题面的样例输入映射成可读变量
2. 帮你确认这个题到底是单值输入、数组输入还是二维输入

所以：

- 读题阶段可以看
- 但不要因为看了 `main` 就跳过自己理解题目
