# SHL External AI Review Brief

## 这份项目本质上在做什么

这不是单纯的 OCR 校对。

这是一个“题库复原 + 题意校正 + 代码验证 + 学习沉淀”的项目：

- 先从不可靠的图片、OCR markdown、同事整理的 docx 里，尽量恢复题目的真实内容。
- 如果题面有明显错误，就根据算法常识、样例、自洽性和代码验证去纠正。
- 如果题面已经残缺到无法稳定复原，就明确定义一个“学习版定稿题面”，保证它自洽、可做、可验证。
- 最终产物不是摘要，而是可读题面 + 可跑 Java 代码，方便后续学英语、练算法、复习实现。

外部 AI 的任务不是“顺着现有结论往下写”，而是批判式复核：

- 现有题面是否自洽
- 现有代码是否匹配题面
- 我们自己定义的“学习版定稿”是否合理
- 哪些题还存在更好的解释

## 你要重点做什么

### 1. 先看题面是否自洽

逐题检查：

- Question
- Input
- Output
- Constraints
- Example
- Explanation

这些是否互相匹配。

### 2. 再看代码是否真的匹配题面

不要只看代码能不能过样例，而要看：

- 代码采用了什么题意解释
- 这个解释是不是最合理的
- 如果不是，更合理的解释是什么

### 3. 特别注意“学习版定稿题”

对于下列题，不要只判断“像不像原题”，还要判断“作为学习版定稿是否自洽、是否值得保留”：

- `A25`
- `B03`

如果你认为我们的定稿不够好，请直接给出你更认可的版本。

### 4. 如果你建议改题面，请给完整题面

不要只给摘要。

请尽量按原题信息密度给：

- Question
- Input
- Output
- Constraints
- Example
- Explanation

### 5. 如果你建议改代码，请明确说清楚

请指出：

- 当前代码采用了什么解释
- 为什么这个解释不够稳
- 你更认可什么解释
- 如果按你的解释，代码应该怎么改
